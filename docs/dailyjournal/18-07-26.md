# Daily Design Journal · Day 16

## Section 1 – Specific Bug

Today's focus was recording system design videos and implementing the storage schema and text processing filters for the indexer library.

The main challenge was designing a highly normalized database schema for the inverted index that prevents dictionary bloating from duplicate strings. In addition, raw document extraction presented parser quirks—specifically, HTML character entities like single quotes (`&#039;` and `&#39;`) were not decoded, and raw JSON API payloads returned from server endpoints were being tokenized as real website content, which polluted the dictionary.

---

## Section 2 – Failed Attempt

Initially, I planned to store matched words directly alongside URLs as string lists inside a single relational table. However, this redundant storage model would grow exponentially. To index efficiently, we designed a split model containing a dictionary table (`words`), a mapping table (`word_occurrences`), and a forward index (`indexed_pages`).

For entity decoding, my initial parsing method missed numeric HTML single quote formats. This left strings like `user's` encoded inside our tables. I fixed this by extending `TextExtractor::decodeEntities` to match and resolve numeric single quotes. To handle JSON, I added a rule in the text extractor to look at the first non-whitespace character; if it meets a brace `{` or `[`, it ignores it and returns empty HTML so the indexing engine ignores data payloads.

## Section 4 – Learning Reflection

Today illustrated that text processing requires strict garbage-in, garbage-out sanitization. Indexing raw data from web crawls without decoding character entities and ignoring API payloads results in corrupt dictionaries.

Furthermore, separating our database components into a dictionary index and a frequency mapping table optimizes space and speeds up B-Tree lookups. By the end of the day, our text extraction parser was fully robust against JSON/entities, the stopword filters were operational, and the database schema was implemented.
