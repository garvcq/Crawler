### Session 1
* Date: 20-07-2026 (12.00AM-2.00AM)

* Duration: 2 hours

* Goal: Assemble the background Indexer runner (`indexer.exe`) and implement the search frequency query CLI (`search.exe`).

* Problem Encountered:
  - Inserting index occurrences row-by-row into the database created a major transaction bottleneck.
  - Retrieving word occurrences to rank results by frequency required efficient queries that join multiple tables to show URLs and Titles quickly.

* Outcome:
  - Assembled the `Indexer` cycle coordinator which polls for unindexed pages, processes them, and writes terms/occurrences inside robust MySQL transactions (`START TRANSACTION`/`COMMIT`) to ensure consistency and speed.
  - Created `search.exe` query processor, which accepts a term, trims/lowercases it, filters out stopwords, and executes a database `JOIN` query including `indexed_pages` to display the top 5 containing URLs, matching frequencies, and page titles.
  - Updated the overall CMake configuration to compile the indexer and search binaries successfully.
