### Session 2
* Date: 18-07-2026 (2.30AM-6.30PM)

* Duration: 4 hours

* Goal: Implement the database storage schema and core text processing libraries for the HTML Indexer.

* Problem Encountered:
  - The indexer needs to store text in a highly normalized structure. Designing the schema as a fast, B-Tree-indexed dictionary required mapping terms to unique IDs in a separate table (`words` and `word_occurrences`) to minimize duplication.
  - Incoming pages contained various HTML entities (e.g., single quotes `&#039;`, `&#39;`) and raw JSON responses which needed to be decoded or ignored to avoid indexing garbage text.

* Outcome:
  - Design and implement `IndexerStorage` with tables `words` (dictionary), `word_occurrences` (inverted index), and `indexed_pages` (forward index).
  - Created `TextExtractor` to strip tags, isolate page titles, decode HTML entities (handling single quotes `&#039;` / `&#39;`), and filter out raw JSON API payloads from entering the index.
  - Developed the custom `Tokenizer` and `StopwordFilter` (using a custom `HashMap` to index/exclude terms from `config/stopwords.txt`).
